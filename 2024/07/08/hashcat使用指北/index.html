hashcat使用指北

常配合john的脚本使用

常用参数

-m 破解hash类型

-a 破解模式

-increment

启用增量破解模式，让hashcat在指定的密码长度范围内执行破解

-increment-min

密码最小长度，后面直接等于一个整数即可，配置increment模式一起使用

-increment-max

密码最大长度，后面直接等于一个整数即可，配置increment模式一起使用

-force

忽略破解过程中的警告信息

-remove

删除已被破解成功的hash

-username

忽略hash文件中的指定的用户名，在破解Linux系统用户密码hash会用到

-potfile-disable

不在potfile中记录破解成功的hash

-O

--optimized-kernel-enable启动优化的内核（限制密码长度）

-d

--opencl-device指定opencl的设备

-D

--opencl-device-types指定opencl的设备类型

一般常用-D 2指定GPU破解

--stdout 不破解哈希，只打印候选项
--show 将哈希列表与potfile进行比较；显示已破解的哈希
--left 将哈希列表与potfile进行比较；显示未破解的哈希
--username 启用忽略哈希文件中的用户名
--remove 启用在破解哈希后删除它们
--remove-timer 数字 每X秒更新输入哈希文件 --remove-timer=30
--potfile-disable 不写入potfile
--potfile-path 文件 potfile的特定路径 --potfile-path=my.pot
--encoding-from 编码 强制将内部词表编码从X --encoding-from=iso-8859-15
--encoding-to 编码 强制将内部词表编码为X --encoding-to=utf-32le
--debug-mode 数字 定义调试模式（仅使用规则进行混合） --debug-mode=4
--debug-file 文件 调试规则的输出文件 --debug-file=good.log
--induction-dir 目录 指定要用于环回的引导目录 --induction=inducts
--outfile-check-dir 目录 指定要监视平原的输出文件目录 --outfile-check-dir=x
--logfile-disable 禁用日志文件
--hccapx-message-pair 数字 仅加载与X匹配的hccapx消息对 --hccapx-message-pair=2
--nonce-error-corrections 数字 替换AP的随机数最后字节的BF大小范围 --nonce-error-corrections=16
--keyboard-layout-mapping 文件 特定哈希模式的键盘布局映射表 --keyb=german.hckmap
--truecrypt-keyfiles 文件 要使用的密钥文件，以逗号分隔 --truecrypt-keyf=x.png
--veracrypt-keyfiles 文件 要使用的密钥文件，以逗号分隔 --veracrypt-keyf=x.txt
--veracrypt-pim 数字 VeraCrypt个人迭代乘数 --veracrypt-pim=1000
-b, --benchmark 运行选定哈希模式的基准测试
--benchmark-all 运行所有哈希模式的基准测试（需要-b）
--speed-only 返回攻击的预期速度，然后退出
--progress-only 返回理想的进度步骤大小和处理时间
-c, --segment-size 数字 设置要从词文件缓存的大小为X -c 32
--bitmap-min 数字 设置位图的最小位数为X --bitmap-min=24
--bitmap-max 数字 设置位图的最大位数为X --bitmap-max=24
--cpu-affinity 字符 锁定到CPU设备，以逗号分隔 --cpu-affinity=1,2,3
--example-hashes 显示每种哈希模式的示例哈希
-I, --opencl-info 显示有关检测到的OpenCL平台/设备的信息 -I
--opencl-platforms 字符 要使用的OpenCL平台，以逗号分隔 --opencl-platforms=2
-d, --opencl-devices 字符 要使用的OpenCL设备，以逗号分隔 -d 1
-D, --opencl-device-types 字符 要使用的OpenCL设备类型，以逗号分隔 -D 1
--opencl-vector-width 数字 手动覆盖OpenCL向量宽度为X --opencl-vector=4
-O, --optimized-kernel-enable 启用优化内核（限制密码长度）
-w, --workload-profile 数字 启用特定的工作负载配置文件，参见下方的池 -w 3
-n, --kernel-accel 数字 手动工作负载调整，设置外循环步长为X -n 64
-u, --kernel-loops 数字 手动工作负载调整，设置内循环步长为X -u 256
-T, --kernel-threads 数字 手动工作负载调整，设置线程数为X -T 64
--spin-damp 数字 使用CPU进行设备同步，以百分比表示 --spin-damp=50
--hwmon-disable 禁用温度和风扇速度读取和触发器
--hwmon-temp-abort 数字 如果温度达到X摄氏度则中止 --hwmon-temp-abort=100
--scrypt-tmto 数字 手动覆盖scrypt的TMTO值为X --scrypt-tmto=3
-s, --skip 数字 从开头跳过X个单词 -s 1000000
-l, --limit 数字 限制为从开头+跳过的单词X个单词 -l 1000000
--keyspace 显示键空间base值然后退出
-j, --rule-left 规则 将单个规则应用于左侧词表中的每个单词 -j ‘c’
-k, --rule-right 规则 将单个规则应用于右侧词表中的每个单词 -k ‘^-‘
-r, --rules-file 文件 将多个规则应用于词表中的每个单词 -r rules/best64.rule
-g, --generate-rules 数字 生成X个随机规则 -g 10000
--generate-rules-func-min 数字 强制每条规则的最小X个功能
--generate-rules-func-max 数字 强制每条规则的最大X个功能
--generate-rules-seed 数字 强制RNG种子设置为X
-1, --custom-charset1 CS 用户定义的字符集?1 -1 ?l?d?u
-i, --increment 启用掩码增量模式
--increment-min 数字 从X开始掩码增量 --increment-min=4
--increment-max 数字 在X停止掩码增量 --increment-max=8
-S, --slow-candidates 启用较慢（但高级）的候选生成器
--brain-server 启用brain服务器
-z, --brain-client 启用brain客户端，激活-S
--brain-client-features 数字 定义brain客户端功能，见下文 --brain-client-features=3
--brain-host 字符 brain服务器主机（IP或域名） --brain-host=127.0.0.1
--brain-port 端口 brain服务器端口 --brain-port=13743
--brain-password 字符 brain服务器认证密码 --brain-password=bZfhCvGUSjRq
--brain-session 十六进制 覆盖自动计算的brain会话 --brain-session=0x2ae611db
--brain-session-whitelist 十六进制 仅允许给定的会话，以逗号分隔 --brain-session-whitelist=0x2ae611db
